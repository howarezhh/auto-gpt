步骤流

从拿到密钥到正式接入

1

拿到平台 API Key

从管理员或用户侧密钥控制台获取平台 API Key。外部请求统一使用平台密钥，不直接暴露后台调试接口。

2

复制 Base URL 和标准路径

把 OpenAI 兼容入口设为 {{ external_v1_base_url }}，只使用当前正式支持的 /v1/models、/v1/chat/completions、/v1/responses、/v1/responses/{response_id} 和 /v1/responses/{response_id}/cancel；本平台当前提供的是兼容子集，不等同于 OpenAI 全量官方能力。

3

先跑一条最短请求

优先调用 /v1/models 或一条最短文本请求验证鉴权、模型列表和路由链路，再写入业务系统或第三方客户端。

4

按客户端类型填配置

支持 OpenAI 兼容 Base URL 且仅使用本平台 /v1/* 兼容子集的客户端，可以按 OpenAI 兼容方式配置 Base URL、API Key 和模型名。

示例流

先发现模型，再发起请求

Step A · 获取模型列表

curl {{ external_v1_base_url }}/models \
  -H "Authorization: Bearer YOUR_PLATFORM_API_KEY"

Step B · chat/completions

curl {{ external_v1_base_url }}/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_PLATFORM_API_KEY" \
  -d '{
    "model": "gpt-4o-mini",
    "messages": [
      {"role": "user", "content": "你好，请介绍一下你自己"}
    ]
  }'

Step C · responses

curl {{ external_v1_base_url }}/responses \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_PLATFORM_API_KEY" \
  -d '{
    "model": "gpt-4o-mini",
    "input": "你好，请用一句话介绍本平台"
  }'

/v1/responses 会优先按原始 Responses 协议透传到命中的上游；只有当上游明确返回端点或模型兼容性错误时，平台才会尝试一次安全互转 fallback。

当前仅简单文本、普通图片和基础流式请求允许进入互转；这类 fallback 可安全适配字段包括 model、instructions、input、temperature、top_p、presence_penalty、frequency_penalty、stream、user、metadata、seed、max_output_tokens、max_tokens。

工具调用、状态上下文、reasoning、结构化输出选项和复杂多模态内容会继续按原请求优先透传；只有在确实需要互转时，平台才会因为存在这些高风险能力而直接返回明确错误码和原因。

错误返回

外部请求一定返回可读结果

JSON 请求错误体

只要平台接收到外部 /v1/* 请求，正常请求返回正常结果；失败请求返回包含错误码、错误说明、追踪 ID 和可重试标记的 JSON。

{
  "error": {
    "message": "具体错误说明",
    "type": "server_error",
    "code": "upstream_request_failed",
    "trace_id": "请求追踪 ID",
    "retryable": true
  }
}

流式请求错误事件

如果 SSE 连接已建立后发生上游中断、超时或空流，平台会发送 event: error，随后发送 data: [DONE] 结束流。

event: error
data: {"error":{"message":"具体错误说明","code":"stream_interrupted"}}

data: [DONE]

端点适配错误

当请求包含不适合在 chat/completions 与 responses 间互转的能力时，平台会返回 endpoint_fallback_conversion_unsafe 或 endpoint_response_conversion_unsafe，并列出不转换原因。

{
  "error": {
    "message": "不安全转换说明",
    "code": "endpoint_fallback_conversion_unsafe",
    "detail": {
      "unsafe_fields": ["tools"],
      "unsafe_reasons": ["工具调用不能安全互转"]
    }
  }
}

Codex / CLI / 自建服务

本地开发工具链、服务端代理或脚本任务只要支持 OpenAI 兼容 Base URL，就可以优先按本平台已开放的 /v1/* 入口直接接入；工具调用、状态上下文、reasoning、多模态等能力是否最终可用，仍取决于目标模型与命中的中转站是否原生支持。

OPENAI_API_KEY=YOUR_PLATFORM_API_KEY
OPENAI_BASE_URL={{ external_v1_base_url }}

Claude Code 官方主要面向 Anthropic Messages API。本平台不提供 Anthropic Messages API 兼容入口；若通过支持 OpenAI 兼容 provider 的接入方式使用，则需确认客户端只依赖当前已开放的 OpenAI 兼容路径、工具能力和模型能力。
