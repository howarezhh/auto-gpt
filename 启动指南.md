查看状态：

```bash
sudo systemctl status aotu-gpt
sudo systemctl status nginx
sudo journalctl -u aotu-gpt -f
```

说明：

- 项目默认通过 `nginx` 对外监听 `80` 端口，后端 `uvicorn` 仅监听 `127.0.0.1:8000`。
- 阿里云安全组只需要放行 `80/TCP`；`8000` 不需要暴露到公网。
- `start_aliyun.sh` 会自动跳过已安装的系统依赖、已存在的虚拟环境和未变更的 Python 依赖。
- `start_aliyun.sh` 会自动补齐生产环境密钥，并在 `EXTERNAL_BASE_URL` 为空时尝试写入公网 IP。
- 部署脚本执行完成后会自动校验 `http://127.0.0.1:8000/login`，并在可识别公网地址时校验公网 `/login`。
- 若需要文档页固定显示公网接入地址，请在 `.env` 中设置 `EXTERNAL_BASE_URL=http://你的公网IP` 或正式域名。



在阿里云服务器上进入项目目录后，常用命令是：

cd /opt/auto-gpt

# 重启后端服务
sudo systemctl restart aotu-gpt

# 如果你改过 nginx 配置，再重启 nginx
sudo systemctl restart nginx

# 查看服务状态
sudo systemctl status aotu-gpt
sudo systemctl status nginx

# 实时看后端日志
sudo journalctl -u aotu-gpt -f

如果你怀疑环境、依赖、.env、systemd 配置有变化，或者想“重新执行整套部署”，可以直接再跑一次脚本；这个脚本本身就是按“可重复执行”写的，已有内容会尽量跳过：

cd /opt/auto-gpt
sudo ./start_aliyun.sh

按当前项目规范，标准建议是：

日常重启服务：sudo systemctl restart aotu-gpt
改了反向代理配置时：再执行 sudo systemctl restart nginx
改了依赖、部署配置或怀疑环境异常时：sudo ./start_aliyun.sh
